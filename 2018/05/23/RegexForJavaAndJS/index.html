什么是正则表达式

正则表达式是由一个字符序列形成的搜索模式，当你在文本中搜索数据时，你可以用搜索模式来描述你要查询的内容。正则表达式可以是一个简单的字符，或一个更复杂的模式。正则表达式可用于所有文本搜索和文本替换的操作。

大部分编程语言都支持正则表达式，每种语言又有细节上的差异，本文只对 Java 和 JavaScript 的正则表达式做一个总结。

JavaScript

在 JavaScript 中，正则表达式的形式是：/表达式/[修饰符]，其中表达式是主体，修饰符可以省略。

  • i：忽略大小写
  • g：全局匹配
  • m：多行匹配

属性

RegExp 对象有几个内置属性：

  • global：该对象是否开启全局匹配模式，布尔值
  • ignoreCase：该对象是否开启忽略大小写模式，布尔值
  • multiline：该对象是否开启多行模式，布尔值
  • lastIndex：该对象下一次开始匹配的位置，即上一次匹配结束的位置，正整数
  • source：该对象的源文本，不包含修饰符，字符串

6、元字符

  • 匹配 0 次或多次，等价于{0,}
  • 匹配 1 次或多次，等价于{0,}
    ? 匹配 0 次或 1 次，等价于{0,1}，如果跟在 *、+、?、{} 的后面匹配会变为惰性模式（匹配尽量少的字符），默认匹配为贪婪模式（匹配尽量多的字符）
    \ 视为转义字符，加在元字符前面可以使其失效
    \d 匹配数字，等价于[0-9]
    \D 匹配非数字，等价于[^0-9]
    ^ 开头，$ 结尾
    . 匹配除换行符以外的任何单个字符
    | 或，满足其中任意一个条件都进行匹配
    x{n} 匹配字符的长度，括号里面是正整数，匹配前一个字符刚好发生了 n 次
    x{n,} 匹配字符的长度，括号里面只有一个正整数，匹配前一个字符至少 n 次
    x{n,m} 匹配字符的长度，括号里面有两个正整数，匹配前一个字符至少 n 次，至多 m 次
    (x) 捕获分组，一般用于提取字符串，返回一个数组，先按照普通字符匹配，如果匹配成功则继续匹配括号中的字符（在上一次匹配结果中进行匹配，每个括号为一组），可以嵌套
    (?:x) 非捕获分组，和上面的相反，不取括号中的匹配结果，可以把一串字符串归为一组，对于像 {n} 这样的只对前一个字符生效的，如果加了 ()，但又不希望分组，则使用这个
    m(?=x) 正向前瞻，只有在 m 后面跟了 x 才能被匹配到 m
    m(?|x) 反向前瞻，只有在 m 后面没有跟 x 才能被匹配到 m，前瞻不消耗字符，只匹配满足前瞻表达式的字符，而不匹配其本身
    [a-zA-Z0-9] 匹配字符的范围，用破折号表示范围
